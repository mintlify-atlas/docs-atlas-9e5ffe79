---
title: "Payments API — Simulated Payment Processing"
description: "Process payments for orders using simulated card, PSE, or cash on pickup methods. Automatically approves most cards; use card ending 0000 to test rejection."
---

The Payments API processes payments for orders using a simulated payment engine — no real card processing occurs. CASH_ON_PICKUP orders are always approved instantly; card and PSE payments simulate approval or rejection based on the card number provided.

## Payment simulation rules

| Method | Behavior |
|--------|----------|
| `CASH_ON_PICKUP` | Always **APPROVED** immediately |
| Any card not ending in `0000` | Always **APPROVED** |
| Card ending in `0000` | Always **REJECTED** |
| `PSE` | Always **APPROVED** |

## Payment object

<ResponseField name="id" type="string">Unique payment ID.</ResponseField>
<ResponseField name="orderId" type="string">Associated order ID.</ResponseField>
<ResponseField name="userId" type="string">Firebase UID of the payer.</ResponseField>
<ResponseField name="amount" type="number">Amount charged.</ResponseField>
<ResponseField name="method" type="string">Payment method used.</ResponseField>
<ResponseField name="status" type="string">One of: `PENDING`, `APPROVED`, `REJECTED`, `REFUNDED`.</ResponseField>
<ResponseField name="simulatedCardLast4" type="string">Last 4 digits of the simulated card (for card payments).</ResponseField>
<ResponseField name="createdAt" type="integer">Epoch ms.</ResponseField>
<ResponseField name="updatedAt" type="integer">Epoch ms.</ResponseField>

## POST /api/v1/payments

Creates and processes a payment for an order. The order must belong to the authenticated user and must not already be paid.

**Auth required:** Yes — **CUSTOMER**

<ParamField body="orderId" type="string" required>ID of the order to pay for.</ParamField>
<ParamField body="method" type="string" required>Payment method: `CREDIT_CARD`, `DEBIT_CARD`, `PSE`, or `CASH_ON_PICKUP`.</ParamField>
<ParamField body="simulatedCardLast4" type="string">Last 4 digits of card (required for card methods). Use `0000` to simulate rejection.</ParamField>

```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"orderId":"ord789","method":"CREDIT_CARD","simulatedCardLast4":"1234"}' \
  http://localhost:8080/api/v1/payments
```

**Approved response (HTTP 200):**
```json
{
  "success": true,
  "data": {
    "id": "pay001",
    "orderId": "ord789",
    "status": "APPROVED",
    "method": "CREDIT_CARD",
    "amount": 7.0
  }
}
```

**Rejected response (HTTP 402):**
```json
{
  "success": false,
  "data": {
    "id": "pay002",
    "orderId": "ord789",
    "status": "REJECTED",
    "method": "CREDIT_CARD",
    "amount": 7.0
  },
  "message": "Pago rechazado. Verifica los datos de tu tarjeta."
}
```

<Note>When a payment is approved, the baker receives an FCM notification for non-CASH orders. When rejected, the customer receives a rejection notification and the response HTTP status is 402 Payment Required.</Note>

## GET /api/v1/payments/order/{orderId}

Returns the payment record for an order.

**Auth required:** Yes (any authenticated user)

<ParamField path="orderId" type="string" required>Order ID.</ParamField>

```bash
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/payments/order/ord789
```

---

## POST /api/v1/payments/order/{orderId}/refund

Refunds a payment. Sets both the payment and order to `REFUNDED` status.

**Auth required:** Yes — **ADMIN**

<ParamField path="orderId" type="string" required>Order ID to refund.</ParamField>

```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/payments/order/ord789/refund
```
